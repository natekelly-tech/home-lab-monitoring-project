### 7. systemd Service Created
Configured the Flask API as a systemd service so it starts automatically on every boot:

**Service file:** `/etc/systemd/system/homelab-monitor.service`

```ini
[Unit]
Description=Homelab Monitor Flask API
After=network.target

[Service]
User=nate
WorkingDirectory=/home/nate/lab/monitoring-api
Environment="PATH=/home/nate/lab/monitoring-api/venv/bin"
ExecStart=/home/nate/lab/monitoring-api/venv/bin/python3 app/main.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Enabled and started with:
```bash
sudo systemctl daemon-reload
sudo systemctl enable homelab-monitor
sudo systemctl start homelab-monitor
```

### 10. Ping Check Bug Identified and Fixed
**Bug:** Ping checks for Main PC, Lab PC, and Ubuntu Server were all returning `down` despite the devices being reachable.

**Investigation steps:**
- Confirmed ping worked from Ubuntu command line ✅
- Confirmed ping worked from Python directly ✅
- Added a `/debug/ping/<host>` route to Flask — confirmed ping worked inside Flask too ✅
- Inspected `checker.py` source code — found the issue

**Root cause:** A typo in `checker.py` — the ping binary path was `/user/bin/ping` instead of `/usr/bin/ping` (missing `s` in `usr`).

**Fix:**
```python
# Before (broken)
["/user/bin/ping", "-c", "1", "-W", "3", host],

# After (fixed)
["/usr/bin/ping", "-c", "1", "-W", "3", host],
```

**Result:** All 6 services immediately returned `up` after fix.

### 11. Debug Route Removed
Removed the temporary `/debug/ping/<host>` route from `main.py` after the bug was resolved, keeping the codebase clean.
