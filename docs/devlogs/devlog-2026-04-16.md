## Issues Encountered

| Issue | Cause | Resolution |
|-------|-------|------------|
| `git commit` failed | Git identity not configured | Set `user.email` and `user.name` globally |
| `git push` rejected | Remote had changes not in local copy | Ran `git pull --rebase` then pushed |
| CHANGELOG corruption | PowerShell `>>` operator wrote wrong encoding | Rewrote entire file using `Set-Content -Encoding UTF8` |
| APK "could not reach API" | Android blocking HTTP cleartext traffic | Moved `usesCleartextTraffic` to `expo-build-properties` plugin |
| `usesCleartextTraffic` schema error | Expo no longer accepts it directly under `android` in app.json | Used `expo-build-properties` plugin instead |

## Key Concepts Learned
- **Why separate repos** — backend and mobile app are distinct codebases with different dependencies and build systems
- **Git workflow** — stage → commit → push, and how to recover from rejected pushes with `git pull --rebase`
- **Development vs production** — code is written on Windows, runs on Ubuntu Server; this mirrors real-world workflows
- **Android HTTP restrictions** — Android requires HTTPS by default; cleartext traffic must be explicitly enabled for local HTTP APIs
- **Expo build properties** — how Expo manages native Android/iOS config through plugins rather than direct manifest edits
